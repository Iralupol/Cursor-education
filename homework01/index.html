Шпаргалка за командами гіта.

  • git config --global user.name - Ім'я користувача потрібно, щоб прив'язувати коміти до вашого імені
  • git config --global user.email - Прив'язання електронної пошти до комітів
  • git init - Ініціалізація репозиторію
  • git status - Перевірка статусу репозиторію
  • git add - Додавання всіх файлів та папок в цю область
  • git commit - Внесення змін до однорядкового повідомлення або через редактор
  • git log - Перегляд історії коммітів із змінами
  • git checkout - Скасування підготовлених та непідготовлених змін
  • git branch - Створення нової гілки та перехід до неї
  • git push - Відправлення змін до віддаленого репозиторію
  • git pull - Отримання змін із віддаленого репозиторію
  • git merge - Злиття віддаленого репозиторію з локальним
  • git clone - Скачує репозиторій разом з всією його історією і змінами

.gitignore файл

Файл із розширенням файлу GITIGNORE - файл Git Ignore, який використовується з системою керування версією / джерелом під назвою Git. Він визначає, які файли та папки не слід ігнорувати у певному вихідному коді.

  • git revert - Вона створює новий коміт, який вносить зміни, протилежні до зазначеного комміту, по суті скасовуючи його.
  • git reset - це потужний спосіб скасувати операцію.
  • git rebase -це процес переміщення чи об'єднання послідовності коммітів у новий базовий комміт
  • git cherry-pick - бере зміни, що вносяться одним коммітом, і намагається повторно застосувати їх у вигляді нового комміта в поточній гілці.
  • git stash - бере незафіксовані зміни (як поетапні, так і не поетапні), зберігає їх для подальшого використання, а потім повертає їх із вашої робочої копії.
